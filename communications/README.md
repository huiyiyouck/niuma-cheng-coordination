# communications — 项目间沟通文档

本目录承接 niuma-cheng 生态内**多个项目之间**的沟通文档。随着生态扩展，会有多份文档共存，每份对应一组参与项目。

## 命名规则

- 两个项目：`{project-a}__{project-b}.md`，如 `xiaobao__ai.md`
- 三个及以上：`{project-a}__{project-b}__{project-c}.md`
- 项目 id 用 [../PROJECTS.md](../PROJECTS.md) 中的稳定短名
- 文件名只表达**参与方**，不表达临时议题；议题写进文档内章节

## 每份沟通文档的两个作用

| 区 | 作用 | 内容 |
|----|------|------|
| **A 需求沟通** | 一方提需求、另一方承接 | 需求台账：提出方 / 内容 / 承接方 / 对应契约 / 状态 |
| **B 联调沟通** | 需求确立后的持续对接 | 接口对接、字段对齐、调试问题、版本跟进（倒序时间线） |

涉及接口/字段的需求，其契约以 [../contracts/](../contracts/) 为单一真源；沟通文档只引用，不复制契约正文。
